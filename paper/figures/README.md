# Figures

All figures are generated by the matplotlib scripts in
`paper/experiments/plot/` from the measured CSVs in
`paper/experiments/results/` (see `paper/notes/experiments_summary.md` for
the underlying experiment setups and takeaways). Regenerate everything with:

```
bash paper/experiments/plot/run_all_plots.sh
```

Style: colorblind-safe Okabe–Ito palette, consistent font sizes, `tight_layout`
(or an equivalent hand-tuned/`bbox_inches="tight"` layout where a shared
colorbar or off-axes legend is present), every axis labeled with units.
Shared style/IO helpers live in `paper/experiments/plot/style.py`.

| Figure | CSV(s) | Script | Shows |
|---|---|---|---|
| `fig_e1_f1_vs_time.{pdf,png}` | `e1_static_quality.csv` | `plot_e1_f1_vs_time.py` | Static-set F1 vs. window index (time), one panel per backend (grid2d, voxel3d), one line per transient-mover density (low/med/high). Both backends recover F1≈1.0 within ~3 windows; heavy clutter (100 movers/window) is the only regime with a visible precision/F1 dip, worse for grid2d than voxel3d. |
| `fig_e2_periodicity.{pdf,png}` | `e2_summary.csv`, `e2_amplitude_vs_length.csv` | `plot_e2_periodicity.py` | Left: Periodic-class TPR (2/3) and FPR (1/5) bars. Right: FreMEn dominant-harmonic amplitude vs. observation length for all 8 probe cells, with the `a_min=0.3` classification threshold and the `T=8`-window amplitude-validity gate marked — shows the two 50%-duty doors clearing the threshold cleanly, the low-duty (25%) door pruned before it matures, and one aperiodic mover producing the false positive. |
| `fig_e3_sensitivity.{pdf,png}` | `e3_sensitivity.csv` | `plot_e3_sensitivity.py` | Three heatmap panels (one per `survival_decay` ∈ {0.90, 0.97, 1.0}); rows = `graduate_prob`, columns = hysteresis-band configuration (no hysteresis / demote=0.5 / demote=0.3, increasing band width). Each cell is colored by flicker-transition count (log scale) and annotated with the raw count and final Static F1 — makes explicit that removing hysteresis (degenerate column) is what drives both catastrophic flicker (574 at the worst config) and F1 loss. |

## Notes on reproducibility

- Every stochastic CSV (E1–E3) was generated with `std::mt19937` seeded from
  the fixed constant `kSeed = 12345` (see each CSV's `#seed` comment row,
  which the plot scripts' `read_csv()` helper skips automatically).
- E4's absolute µs/updates-per-second figures are machine-dependent (per
  `notes/experiments_summary.md`); the cross-backend/size **ratios** and the
  live-cell counts are the reproducible quantities and are what
  `fig_e4_throughput` is meant to convey.
